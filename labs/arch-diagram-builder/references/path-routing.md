# 连线布线：避免箭头堆叠

双向或多条连线经过同一区域时，**禁止共用同一条 path 或同一锚点**。这是架构图可读性的关键。

## 原则

1. **分车道**：平行双向线垂直偏移 ≥ 40px（推荐 50px+）
2. **分弧线**：同一对节点的去程/回程走不同侧（左弧 vs 右弧）
3. **分锚点**：同一节点多条出线从 rect 不同边/不同 y 出发
4. **绕开枢纽**：星型节点（如 Agent Runtime）各方向出口分散到四边

## 双向对常用策略

| 关系 | 去程 | 回程 |
|------|------|------|
| A ↔ B 水平 | 上轨 `y = center - 20` | 下轨 `y = center + 32` |
| A ↔ B 斜向 | 左侧大弧线（控制点偏左） | 右侧大弧线（控制点偏右） |
| A ↔ B 垂直 | 从 A 底中 → B 顶中 | 从 B 底右 → A 顶右（x 偏移） |

## Agent 开发系统实测锚点（参考）

```
Runtime ↔ LLM:
  去程: M 505 200 L 640 200      (推理请求，上轨)
  回程: M 640 252 L 505 252      (计划/动作，下轨)

Runtime ↔ Tools:
  去程: M 385 258 C 260 300 170 328 130 350   (左弧向下)
  回程: M 238 350 C 360 305 455 278 478 258   (右弧向上)

Tools ↔ External:
  去程: M 110 406 C 75 418 62 432 72 440      (贴左下)
  回程: M 168 440 C 205 422 218 408 228 406   (贴右上)

Memory → Context（绕远）:
  M 388 408 C 220 408 90 310 88 257           (左侧远弧，不穿过 Runtime)

Runtime → Memory（独立竖道）:
  M 452 258 L 452 400                         (x=452 竖线，与 Context 回路分离)

Response → API（顶车道）:
  M 640 94 L 540 94                           (y=94，避开 Runtime→Response 弧线)
```

## 标签放置

- 标签放在**车道中间偏空处**，不要压在节点上
- 双向标签分别靠近各自车道（上轨标签 y 更小）
