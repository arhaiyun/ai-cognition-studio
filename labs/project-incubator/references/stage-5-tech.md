# 技术实现方案

## 1. 技术栈选型
| 维度 | 选择 | 一句话理由 | 替代方案（脚注） |
| 前端框架 | ... | ... | |
| 状态管理 | ... | ... | |
| UI 组件库 | ... | ... | |
| 后端框架 | ... | ... | |
| 数据库 | ... | ... | |
| 缓存 | ... | ... | |
| 部署 | ... | ... | |
| 监控 | ... | ... | |

## 2. 前端架构
- 路由策略
- 状态管理分层（local / shared / server）
- 组件分层（atoms / molecules / pages）
- 数据请求与缓存策略
- 错误边界与降级

## 3. 后端架构
- 分层（controller / service / repo / model）
- 鉴权流程
- 异步任务
- 第三方集成点

## 4. 数据库设计
- 所有 entity 的 schema（DDL 级别）
- 索引策略
- 关系图（HTML+SVG，存到 `tech-assets/erd.html`）

## 5. API 设计
- RESTful 或 RPC 选择 + 理由
- 所有 endpoint 列表：method / path / 入参 / 出参 / 状态码 / 错误码
- 鉴权 / 限流 / 幂等性策略

## 6. 目录结构
完整树状（含关键文件名）

## 7. 核心模块拆分
每个模块：职责 / 对外接口 / 依赖 / 测试策略

## 8. 状态管理方案
具体到状态片段如何切分、哪些常驻、哪些临时

## 9. 权限认证方案
- 认证方式（JWT / Session / OAuth）
- 权限校验时机（middleware / decorator / 业务层）
- 前后端如何同步权限状态

## 10. 部署方案
- 环境：dev / staging / prod
- CI/CD 流程
- 域名 / HTTPS / CDN

## 11. 测试方案
- 单元测试覆盖目标
- 集成测试关键流程
- E2E 测试 happy path

## 12. 开发里程碑
按"可独立验证的最小单元"切分，每个里程碑写：
- 范围 / 工期估算 / 验收方式

## 13. 风险与取舍
- 技术风险 + 应对
- 性能风险 + 应对
- 成本风险 + 应对
- 三个"如果重来一次会怎么改"的诚实反思
