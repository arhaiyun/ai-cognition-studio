# Stage 4: UED HTML 交互 Demo

**只有 PRD 确认后才能做。**

不是静态截图，是**可在浏览器打开、能点击切换页面、能看到状态变化的真实交互 Demo**。

## Step 4.1: 风格锚点

先列 3 个**真实产品**作为视觉/交互参考：

```
- 候选 A：<产品名> — <调性一句话> — 匹配度 X/10 — 理由
- 候选 B：...
- 候选 C：...

选定主基调：<候选 X>
理由：<为什么这个调性匹配目标用户>
风格关键词（3-5 个具体词）：例如"信息密度高 / 等宽数字 / 负空间克制"
禁用："现代 / 简洁 / 高端" 这种万能词
```

## Step 4.2: 设计系统

```
- 色板：所有 hex 值（主色 / 辅助 / 语义色 success/warning/error / 中性灰阶 9 档）
- 字体：主字体（中英文各一个）/ 等宽字体（用于数字）/ 字号体系（具体 px 或 rem）
- 间距：4pt 或 8pt grid
- 圆角：sm/md/lg 具体值
- 阴影：层级 1/2/3 的 box-shadow 值
- 动效：缓动曲线 + 时长（hover 150ms / panel 250ms 等）
```

## Step 4.3: 信息架构

页面 / 模块树。

## Step 4.4: 产出可运行 Demo

**必须产出物**（写到当前工作目录）：

```
demo/
├── index.html              # 入口，含导航
├── pages/
│   ├── <page-1>.html      # 每个核心页面一份
│   ├── <page-2>.html
│   └── ...
├── assets/
│   ├── style.css          # 设计系统 + 组件样式
│   ├── app.js             # 切换 / 模拟数据 / 状态变化
│   └── mock-data.js       # 真实业务示例数据
└── README.md              # Demo 怎么打开 + 每个页面演示什么
```

## Demo 强制要求

- ✅ 真实文案：用真实业务场景写文案，禁止 "Lorem ipsum / User Name / Item 1"
- ✅ 真实数据：mock data 用真实数字、真实姓名、真实场景
- ✅ 真实视觉：禁止灰色方块占位，所有元素都要有内容
- ✅ 表达交互：至少包含 表单 / 列表 / 卡片 / 弹窗 / 下拉 / Tab / 状态切换 各一个
- ✅ 覆盖状态：每个核心页面都要演示 加载中 / 空状态 / 错误状态 / 成功状态
- ✅ 导航可走：从首页能点到每个核心页面，能演示 1-2 个完整 happy path
- ✅ 响应式：至少能在 1440px 桌面 + 375px 移动 两个断点正常显示
- ❌ 不要框架：纯 HTML + CSS + 少量 vanilla JS。可用 Tailwind CDN，但不要 React / Vue
- ❌ 不要后端：所有数据是 mock，前端跑
