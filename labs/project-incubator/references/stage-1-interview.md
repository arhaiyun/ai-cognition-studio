# Stage 1: 需求访谈（多轮澄清）

**禁止跳过此阶段直接写方案。**

如果用户的输入只是"做一个 X"，你必须先组织结构化提问（Claude Code 用 AskUserQuestion；Codex 在对话中列出问题）。

## 必须问到的维度（按 Stage 0 领域调整顺序与权重）

通用必问：

1. **一句话定义**：用户能否用一句话说清这是什么
2. **目标用户**：具体到一个人物画像（年龄/职业/场景），不要"所有人"
3. **核心痛点**：现在用户用什么替代方案，有什么不满
4. **使用场景**：什么时间/地点/情境下用
5. **平台形态**：Web / 移动 App / 小程序 / Chrome 插件 / 桌面 App / CLI / API 服务 / 后台
6. **目标**：商业目标 or 个人目标 or 学习目标
7. **竞品/参考**：有没有"对标"产品（关键，用来定风格锚点）
8. **风格偏好**：用户审美倾向（极简 / 信息密度高 / 拟物 / 工程师风 / etc.）
9. **技术栈约束**：用户偏好/必须用某栈，或完全开放
10. **资源约束**：上线时间 / 预算 / 团队规模 / 是否独立开发

**领域专属问题**（按 Stage 0 列出的深度维度，每个维度至少 1 个问题）。

## 提问方式

- 使用结构化提问，一次最多 4 个问题（Claude Code 用 AskUserQuestion；Codex 在对话中列出带选项的问题）
- 每个选项都要给 Recommended 默认推荐
- 优先问"会改变架构 / 范围 / 用户群"的高杠杆问题，不要问可以自己拍板的细节
- 如果用户回答模糊，继续追问；不要假设默认值

## STOP 等用户答完所有必要问题，再进 Stage 2
