# Prompt Patterns · 可复用模板

复制到对话或 `.private/scratch/` 使用。按场景替换 `{{占位}}`。

---

## 1. 写技术 Spec / ADR

```
我要做：{{一句话目标}}

请按以下结构输出 Markdown spec：
1. 背景与问题
2. 目标与非目标
3. 方案对比（2–3 个）
4. 推荐方案与理由
5. 风险与开放问题

约束：{{技术栈/时间/不能做什么}}
读者：{{自己/团队/评审者}}
```

---

## 2. Debug（有报错日志）

```
现象：{{用户可见行为}}
期望：{{应该怎样}}
环境：{{OS/版本/相关配置}}

报错/日志：
```
{{粘贴}}
```

请：1) 归纳可能根因（按概率） 2) 最小验证步骤 3) 修复建议（最小 diff）
不要先大重构。
```

---

## 3. Code Review（副驾）

```
请 review 以下 diff，关注：正确性、边界、安全、与现有风格一致性。
不必吹毛求疵风格细节。

重点文件：{{paths}}

输出：
- 🔴 必须改
- 🟡 建议改
- 🟢 可选
- 若无问题明确说 LGTM
```

---

## 4. 从播客/文章提炼

```
来源：{{链接或标题}}
我要沉淀到 ai-cognition-studio。

请输出：
1. TL;DR（3 句内）
2. 三条 takeaway（ actionable）
3. 建议目录：cognition / essays / podcast / labs 之一
4. 可选：一个 mini lab 想法（若能动手验证）
```

---

## 5. MCP 工具设计 brainstorm

```
API/能力：{{简述}}
Agent 典型任务：{{例子}}

请建议：
- 5 个 tool 名称 + 一句话描述
- 哪些适合合并为 workflow tool
- 哪些需要分页参数
- 一个「不要暴露」的反例及原因
```

---

## 维护

新模式以 PR 形式追加；每条注明 **适用场景** 与 **反模式**。
