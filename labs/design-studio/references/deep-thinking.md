# 深度思考：自媒体设计如何高效且专业

## 1. 问题本质

自媒体视觉生产的核心矛盾不是「会不会设计」，而是：

| 矛盾 | 表现 | 后果 |
|------|------|------|
| **创意 vs 重复** | 每篇都要新封面，但系列要统一品牌 | 要么千篇一律，要么风格漂移 |
| **深度 vs 注意力** | 长文有洞见，平台要 3 秒钩子 | 要么浅薄爆款，要么无人看 |
| **静态 vs 动态** | 架构图要步进演示，短视频要 9:16 | 同一内容手工做三遍 |
| **AI 生成 vs 品控** | Agent 秒出图，但箭头堆叠、配色混乱 | 「AI slop」损害专业感 |

**结论**：高效专业的关键不是更强的「审美 prompt」，而是**工程化设计系统**——把不变的东西固化，把变化的东西模板化。

## 2. 三层模型

```
┌─────────────────────────────────────────────────────────┐
│  L3 分发层：平台规格（9:16 / 16:9 / OG 1200×630）        │
├─────────────────────────────────────────────────────────┤
│  L2 形态层：媒介选择（Keynote / 交互图 / 封面 / 视频）    │
├─────────────────────────────────────────────────────────┤
│  L1 品牌层：Token（色、字、水印、语义色、间距）           │
└─────────────────────────────────────────────────────────┘
         ▲                           ▲
         │                           │
    固定（references）          可变（content-template）
```

- **L1 品牌层**：一次定义，全局复用。本项目见 `brand-tokens.md`。
- **L2 形态层**：按「讲观点 / 讲结构 / 讲步骤」选 Skill，见 `output-routing.md`。
- **L3 分发层**：同一内容多平台时，只改编排比例与安全区，不改编叙事结构。

## 3. 效率公式

```
专业产出 = 设计系统（固定）× 内容模板（可变）× Harness（验收）
```

| 因子 | 本项目实践 | 反模式 |
|------|-----------|--------|
| 设计系统 | `arch-diagram-builder/references/design-system.md` | 每次让 AI「设计好看一点」 |
| 内容模板 | `content-template.md`、ppt 分页大纲 | 把整篇文章贴进一页 |
| Harness | `verify-diagram.mjs`、口播时长估算 | 生成完不打开、不步进 |

**10× 效率来源**：不是生成更快，而是**返工更少**。

## 4. 媒介选择决策树

```
内容要表达什么？
├─ 观点 / 框架 / 金句 → ppt-builder（多页叙事）
├─ 系统如何跑 / 数据怎么流 → arch-diagram-builder（单页步进）
├─ 步骤 / 状态 / 时序（文档内） → Mermaid flowchart / sequence
├─ 白板讨论 / 草图感 → Excalidraw
├─ 15–60s 竖屏短视频 → Remotion（代码渲染，可 A/B）
└─ 缩略图 / 封面 / 卡片 → design-studio 封面快路径
```

**不要**用架构图 Skill 做 Keynote，**不要**用 Keynote 做 runtime 步进——工具边界清晰才能沉淀。

## 5. 专业感的六个来源

1. **语义色一致**：蓝=入口、紫=智能、绿=存储…（arch 五色）或 Google 四色（keynote）
2. **顺序可见**：flow-strip、角标、conn-order——观众不读稿也能跟上
3. **布线不堆叠**：双向线分车道（`path-routing.md`）
4. **结论先行标题**：封面和每页标题写答案，不写名词
5. **留白与字号层级**：vw 单位、一页一观点
6. **品牌水印统一**：云牧元宇宙，位置固定

## 6. AI 时代的分工

| 人 | Agent / Skill |
|----|---------------|
| 选题、论点、事实核查 | 套模板、生成 HTML、跑验收 |
| 步进顺序与案例选择 | SVG 布线、粒子动画、CSS |
| 平台标题 / 标签策略 | 多尺寸改编排、manifest 更新 |
| 最终品控（5 秒法则） | Chrome 预览、verify 脚本 |

人负责**什么值得说**；Skill 负责**怎么说才专业**。

## 7. 与「frontend-design」的关系

`frontend-design`（Anthropic）强调每次不同、避免 AI 审美同质化——适合**一次性 landing / poster**。

自媒体系列内容应**优先 design-studio 品牌 token**，仅在「破圈实验封面」时借用 frontend-design 的差异化策略。
