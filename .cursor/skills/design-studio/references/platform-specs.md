# 平台规格

生成前确认目标平台，尺寸与安全区不可随意改。

## 横屏视频 / 演讲

| 平台 | 比例 | 像素 | 备注 |
|------|------|------|------|
| B站 / YouTube | 16:9 | 1920×1080 | 默认 arch-diagram、keynote 全屏 |
| 大会投影 | 16:9 | 1920×1080 | vw 自适应，Chrome 全屏 F |
| 飞书文档嵌入 | 16:9 | iframe 宽度 100% | cognition frontmatter `diagram:` |

## 竖屏短视频

| 平台 | 比例 | 像素 | 安全区 |
|------|------|------|--------|
| 抖音 / TikTok | 9:16 | 1080×1920 | 顶 120px，底 280px，右 80px |
| 小红书视频 | 9:16 | 1080×1920 | 底栏 UI 更高，底 320px |
| 微信视频号 | 9:16 | 1080×1920 | 同抖音 |

竖屏产出优先 Remotion；临时可用 HTML 固定 `aspect-ratio: 9/16` 录屏。

## 静态图 / 封面

| 平台 | 像素 | 比例 | 路径建议 |
|------|------|------|----------|
| 公众号头图 | 900×383 | ≈2.35:1 | `assets/{系列}/cover-wechat.png` |
| 小红书封面 | 1080×1440 | 3:4 | `assets/{系列}/cover-xhs.png` |
| B站封面 | 1146×717 | 16:10 | `assets/{系列}/cover-bilibili.png` |
| OG / Twitter | 1200×630 | 1.91:1 | `assets/{系列}/og.png` |
| 正方形头像/图标 | 800×800 | 1:1 | `assets/{系列}/square.png` |

HTML 封面页内用 `meta` 标注目标导出尺寸，Chrome 截图或 `verify-pipeline.mjs` 提示。

## 口播时长参考

| 形态 | 时长 | 步数/页数 |
|------|------|-----------|
| 架构图步进 | 4–9 min | 6–9 步 |
| Keynote 分享 | 15–20 min | 12–18 页 |
| 竖屏短视频 | 30–90 s | 3–5 镜头 |
| 投资长视频 | 20–30 min | 4 图 + 4 篇文章 |

## 默认选择（用户未指定时）

- 技术架构 → 16:9 白底交互图 + B站横屏口播
- 观点分享 → 16:9 Keynote
- 投资系列 → 16:9 交互图 + 公众号/飞书长文嵌入
- 破圈短视频 → 9:16 Remotion 切片（第二阶段）
