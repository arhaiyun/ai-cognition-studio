# AI 投资研究 · 下篇：A 股 CSP 产业链与 Agent 兑现

> **TL;DR**：A 股 AI alpha 在 **CSP 资本开支产业链**（光模块、服务器、AIDC、国产算力）+ **能收钱的 Agent 应用**。绑定头部互联网 capex 的供应链，业绩能见度高于纯概念。

**系列**：[上篇](04-ai-investment-part1-seven-layers.md) · [中篇](05-ai-investment-part2-us-equities.md) · **下篇** · [风险](07-ai-investment-part4-risks.md)

---

## 1 · 中国 CSP 格局（资本开支发动机）

| CSP | 特点 | 产业链影响 |
|-----|------|------------|
| **字节跳动** | 国内 capex 激进；豆包/扣子生态 | 服务器、光模块、智算租赁订单集中 |
| **阿里巴巴** | 通义 + 阿里云；开源+云一体 | 算力基建 + 模型商业化双线 |
| **腾讯** | 微信生态 + 混元；社交/游戏/广告场景 | 应用变现强，基建跟投 |
| **百度** | 文心 + 智能云；政企场景 | 大模型+云捆绑 |
| **华为** | 昇腾生态；政企/运营商 | 国产算力集群、全栈替代 |

**A 股逻辑**：不像美股直接买 Mag7，大 A 更多是 **「谁进 CSP 供应链、谁有真实 GPU 上架」**。

## 3 · 2026 A 股两条主线（机构共识摘要）

### 主线 A：算力「全链通胀」

- 训推共振 → 算力缺口  
- 景气从芯片外溢到 **AIDC、液冷、电力、光模块**  
- **选股**：深度绑定 CSP 招标、有在手订单、毛利率改善  

### 主线 B：应用商业化

- Agent、多模态、垂类（医疗、工业、金融）  
- 从主题 → **收入/合同/付费用户**

## 4 · Agent 兑现案例（怎么判断「真 Agent」）

结合本仓库 [AI Agent 开发系统架构](../diagrams/ai-agent-dev-system.html) 与 [Cursor 案例口播](../presentation/ai-agent-dev-system-architecture-script.md)：

| 案例类型 | 场景 | 兑现信号 | A股/HK 映射思路 |
|----------|------|----------|-----------------|
| **Coding Agent** | Cursor 类：读 repo、改文件、跑测试 | 付费订阅、日活开发者、ARPU | 映射开发者工具链、云基础设施（间接） |
| **办公 Agent** | 文档生成+企业知识库 | WPS AI 付费率、企业订单 | **金山办公** |
| **金融 Agent** | 投研/交易辅助 | 付费用户、监管合规落地 | **同花顺、恒生电子** |
| **车载 Agent** | 智驾+座舱大模型 | 车型搭载量、OEM 订单 | **德赛西威** 等 |
| **CSP 原生** | 豆包/通义/文心 API+Agent 平台 | API 调用量、MaaS 收入占比 | 阿里、腾讯、百度（港股） |

### Agent 兑现 vs 概念炒作：5 个验证问题

1. 用户使用时 **AI 是否参与 runtime 执行**？（见 [AI 产品六层](02-ai-product-six-layers.md)）  
2. 是否有 **可追踪的 workflow**（Plan-Act-Observe），不是一次性生成？  
3. 财报是否披露 **AI 相关收入/付费用户/ARPU**？  
4. 是否绑定 **真实工具调用**（API、数据库、交易接口）？  
5. 是否有 **eval / 成功率** 或客户案例可审计？

## 6 · 下篇结论

1. **A 股确定性** 在 CSP 产业链中游（尤其光模块、服务器、AIDC）  
2. **A 股弹性** 在应用层，但必须用 **Agent 兑现五问** 筛掉纯概念  
3. **港股互联网** 是观察 CSP capex 与云变现的「总开关」
