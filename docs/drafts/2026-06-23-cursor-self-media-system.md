# 用 Cursor 建立自媒体视频生产系统

> 真正值得自动化的不是表达本身，而是表达背后那些重复、可验证、可复用的工作。

很多创作者已经会用 Cursor 写选题、查资料和生成脚本，但生产效率并没有因此稳定提升。

问题不在模型能力，而在使用方式：我们把 Cursor 当成随叫随到的写手，每条内容都从一段新对话开始。账号定位要重新解释，资料散落在聊天记录里，脚本与素材彼此脱节，发布数据也不会进入下一次创作。

这种方式优化了某个动作，却没有优化完整流程。

要让 Cursor 真正成为生产力工具，需要完成一次关键转变：

> 从“让 AI 生成内容”，转向“让 Cursor 管理内容的完整生命周期”。

这不是追求全自动创作，而是建立一套能够沉淀判断、复用资产并持续学习的个人内容系统。

## 一、AI 写稿为什么没有带来系统效率

一条视频并不是从脚本开始，也不会在导出视频时结束。它至少包含四类工作：

- **判断**：服务谁、说什么、为什么现在值得说；
- **研究**：事实、案例、争议和论证边界；
- **生产**：脚本、分镜、素材、录制与剪辑；
- **经营**：包装、分发、数据分析与经验沉淀。

单独压缩写稿时间，无法解决阶段之间的信息损耗。研究结论没有进入脚本，脚本中的画面没有变成素材任务，评论区的新问题也没有回到选题池。创作者仍然需要不断搬运信息、补充背景和纠正偏差。

因此，内容生产真正的成本并不只是“写得慢”，而是三个隐藏成本：

1. **上下文重建**：每次都要重新告诉 AI 你是谁、为谁创作；
2. **阶段返工**：上游判断不清，越到后期修改代价越高；
3. **经验流失**：一次创作结束后，方法和数据没有成为下一次的输入。

专业系统首先要降低这三类成本。生成速度只是结果，不是目标。

## 二、核心转变：从任务自动化到生命周期管理

一个成熟的创作系统，不是更长的 Prompt，而是三种机制的组合。

### 1. 用稳定上下文保持方向

账号定位、目标受众、表达风格、事实标准和视觉规范属于长期资产。它们应保存在项目文档中，由 Cursor 按需读取，而不是依赖创作者每次复述。

稳定上下文解决的是“系统是否始终知道你是谁”。

### 2. 用标准产物连接阶段

选题卡、研究包、内容 Brief、视频脚本、素材清单和复盘报告，应该拥有固定结构。上一个阶段的输出，就是下一个阶段的输入。

标准产物解决的是“信息能否可靠地向后流动”。

### 3. 用反馈机制积累能力

发布之后，点击率、前段留存、完播率、评论问题和转化结果必须回到系统。复盘不能停留在“这条表现不错”，而应转化为下一轮可以执行的改变。

反馈机制解决的是“系统能否因为使用而变得更好”。

这三种机制共同构成内容工程化：把稳定的部分固化，把重复的部分自动化，把需要判断的部分留给人。

## 三、系统架构：四层，而不是一个万能 Skill

最容易想到的方案，是创建一个包办选题、研究、写稿、制作和复盘的巨型 Skill。它在初期看似方便，随着规则增加，却会迅速变得不可控：上下文臃肿、职责冲突、输出不稳定，也很难判断问题出在哪一层。

更可靠的结构由四层组成。

### 编排层：管理进度，不代替专业判断

`self-media-studio` 是系统入口。它负责识别当前阶段、检查输入是否充分、调用对应 Skill，并管理中间产物。

它更像制片人，而不是一个万能创作者。

### 能力层：每个 Skill 只解决一类问题

例如：

- `topic-planner`：把灵感收敛为值得投入的选题；
- `content-researcher`：建立事实、案例和争议清单；
- `video-script-writer`：把已确认的内容 Brief 转换成可制作脚本；
- `content-packager`：针对不同平台设计标题、封面信息和发布文案；
- `content-analyst`：将发布数据转化为下一轮行动。

Skill 的价值不在数量，而在边界清楚、输入稳定、输出可交接。

### 资产层：保存创作者真正的差异

账号定位、个人经历、观点、案例、可靠来源、视觉模板和历史复盘，都属于长期资产。

通用模型可以生成流畅表达，却无法凭空拥有创作者的真实经验。内容差异最终来自资产层，而不是来自更复杂的提示词。

### 反馈层：让系统从结果中学习

反馈层连接发布结果与下一轮选题。它记录哪些主题值得继续、哪类开头容易流失、哪些案例带来收藏，以及观众正在重复提出什么问题。

没有反馈层，所谓“内容系统”仍然只是一条更快的流水线。

## 四、一条视频应该如何在系统中流动

假设最初只有一个模糊想法：

> Cursor 会改变个人创作者的生产方式。

系统不应立即生成脚本，而应逐步降低不确定性。

### 第一步：把想法变成选题

先明确受众、问题、内容价值和差异化。经过判断后，选题可以收敛为：

> 多数人把 Cursor 当成写稿工具，但它更大的价值，是帮助个人创作者建立一套可复用的内容生产系统。

此时，主题才拥有明确冲突和观众收益。

### 第二步：把选题变成可信论证

研究不是搜集尽可能多的材料，而是回答支撑结论所必需的问题：

- 视频生产包含哪些关键阶段？
- 单点使用 AI 为什么无法形成长期效率？
- 哪些工作适合交给 Cursor？
- 哪些判断必须由创作者保留？
- 这套方法在哪些场景下并不适用？

研究产物应同时包含证据、来源、反例和未知项。未经确认的事实不能被“顺滑地写进”脚本。

### 第三步：把论证变成内容设计

内容 Brief 要在写稿前确定五件事：

1. 目标受众；
2. 核心结论；
3. 观众看完后的具体收获；
4. 叙事顺序；
5. 计划使用的案例和证据。

Brief 是最重要的人工确认点。方向错误时，在这里修改最便宜。

### 第四步：把内容设计变成制作任务

视频脚本不能只包含口播文字，还应明确时间段、表达目标、画面形式和素材需求。随后由系统提取录屏、图表、字幕卡、封面和 B-roll 等任务。

这一步的核心，是让脚本真正连接制作，而不是成为另一篇文章。

### 第五步：把一次发布变成系统经验

发布后，系统记录点击、留存、评论和转化，并回答：

- 用户为什么点击或没有点击？
- 最大流失发生在哪里？
- 哪个案例真正帮助了理解？
- 评论区出现了哪些新问题？
- 下一条内容具体应该改变什么？

复盘必须产出动作，而不是评价。

## 五、人和 Cursor 的正确分工

内容系统的目标不是消灭人的参与，而是保护人的判断力，把它用在最有价值的位置。

| 人负责 | Cursor 负责 |
|---|---|
| 选择值得讨论的问题 | 整理和检索资料 |
| 提出真正相信的观点 | 检查论证缺口与反例 |
| 提供亲历案例与独特证据 | 把 Brief 转换为标准产物 |
| 决定表达边界和取舍 | 提取素材任务与发布清单 |
| 判断最终成品是否代表自己 | 执行格式检查和机械处理 |
| 解释数据背后的原因 | 汇总数据并发现重复模式 |

可以用一句话概括：

> 人决定什么值得说，Cursor 负责让正确的信息在正确的阶段，以正确的格式出现。

一旦把立场、取舍和最终审美也交给模型，生产速度可能提高，内容却会迅速失去辨识度。

## 六、最小可行系统：先建设六个 Skill

第一版不需要完整覆盖所有环节。最合理的起点，是建立一条能够闭环的最短路径：

1. **`self-media-studio`**：创建项目、识别阶段、管理交接；
2. **`topic-planner`**：完成选题定义与投入判断；
3. **`content-researcher`**：建立可信的研究包；
4. **`video-script-writer`**：输出可直接制作的视频脚本；
5. **`content-packager`**：生成平台适配后的发布包；
6. **`content-analyst`**：把数据转化为下一轮行动。

这六个 Skill 已经覆盖从想法到反馈的闭环。事实核查、视觉设计、代码化视频和自动发布，可以在真实瓶颈出现后再补充。

判断是否应该新增 Skill，可以问三个问题：

- 这个任务是否在多条内容中重复出现？
- 它的输入和输出是否可以被清楚定义？
- 标准化后是否能减少返工或风险？

如果三个答案都是否，就先不要把它封装成 Skill。

## 七、实施原则：先跑通一次，再设计系统

内容系统最危险的误区，是花大量时间搭建一套尚未被真实创作验证的复杂流程。

更有效的实施顺序是：

### 第一轮：人工跑通

选择一个真实选题，依次完成选题卡、研究包、内容 Brief、脚本、发布包和复盘。先观察信息在哪里丢失、哪些动作最重复、返工主要发生在哪一段。

### 第二轮：固化模板

把第一轮中稳定的中间产物整理为模板，使每个阶段都有明确交付标准。

### 第三轮：封装 Skill

只把已经重复出现、边界清楚的流程封装为 Skill。不要自动化尚未理解的工作。

### 第四轮：连接数据

让复盘结论进入选题库、脚本规则和案例库。系统从这一刻开始具备学习能力。

## 结语：把个人判断沉淀成生产能力

Cursor 对创作者最重要的价值，不是一次生成更多文字，而是让个人经验、判断和工作方法有机会沉淀为可复用的系统。

这套系统的专业性不取决于拥有多少 Skill，而取决于四件事：

- 长期上下文是否稳定；
- 阶段之间能否可靠交接；
- 人与 AI 的职责是否清楚；
- 发布结果能否改变下一轮创作。

真正值得追求的，也不是“无人参与的自动创作”。创作的核心仍然是人的观察、立场和取舍。Cursor 应该减少的是重复劳动、信息损耗和低级错误，从而让创作者把更多注意力留给真正重要的问题。

因此，第一步不是建造一座庞大的内容工厂。

第一步是选择一个真实想法，完整走完一次：

> 想法 → 选题 → 研究 → Brief → 脚本 → 发布 → 复盘。

当这条路径真实地跑通，属于你的内容生产系统才开始生长。
