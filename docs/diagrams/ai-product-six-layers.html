AI 产品的六个层次 · 完整架构图

核心 distinction：判断是否为 AI 产品，不看开发时是否用 AI，而看用户使用时 AI 是否参与 runtime（理解、判断、行动、记忆、协作、交付）。

来源：Superlinear Academy · AI产品的六个层次 · 对齐 Anthropic / OpenAI / Google Cloud agent 框架

0 · 先区分两件事

AI-assisted Building

AI 在开发过程中帮你写代码、造软件。用户打开产品时，AI 不参与 runtime。

例：用 Cursor 写的普通日历 App

≠ AI 产品

AI Runtime · AI Product

AI 在产品运行过程中读取上下文、调用工具、做决策、执行任务、记忆与交付。

例：邮件助手读信 → 判断意图 → 调 Gmail API → 等人确认 → 发送

I · AI 应用六层架构（自下而上 · 成熟度递增）

L1

Prompt Wrapper · 薄 AI 工具

Model as a feature — 模型只是文本处理功能

用户输入 → prompt template → LLM API → 输出 → 展示
标题/邮件/简历生成 无上下文 无工具 ≈ L4 早期 demo
L2

Grounded AI / RAG · 带知识的 AI

AI knows more, but still mostly answers

问题 → query rewrite → 检索 KB/DB → rerank → 拼接上下文 → LLM → citation
chunking / embedding hybrid search permission-aware retrieval 知识库/客服 FAQ
L3

Tool-using AI · 能调用工具

Model as an interface to tools — 模型成为工具入口

用户目标 → LLM 选工具 → function call / API → 获取结果 → 继续执行
tool schema auth / permission confirmation Gmail / CRM / 日历
L4

LLM Workflow · 固定流程里的 AI

AI inside a controlled process — 可控业务流程

分类 → 检索 → 抽取 → 调 API → 生成方案 → 人工确认 → 执行 → 日志/eval
Anthropic Workflow 可预测性 human-in-the-loop ≈ L5 主战场
L5

Agentic Core · 智能体核心

Model as a controller — 模型控制任务执行

Goal → Plan → Choose tool → Act → Observe → Update state → Continue / Stop / Ask human
ReAct loop tool registry guardrails evaluator / tracing ≈ L6 底层能力
L6

AI-native Product / System · 落地的 AI 系统

AI 不再是一个功能，而是系统的智能层 · Building what you can't buy

Agentic Core + 产品场景 + 三大能力模块（见下方展开）
低摩擦交互 上下文智能 主动触发 权限 / eval / runtime ≈ L6 Architect 完整形态

II · L6 系统三大产品能力 → 技术模块

Frictionless Interaction
低摩擦交互

  • chat / voice / command palette
  • browser extension / inline edit
  • sidebar / one-click apply
  • approval UI · undo / review
  • human-in-the-loop

Contextual Intelligence
上下文智能

  • user profile · workspace context
  • conversation state · documents
  • emails / calendar / CRM / tickets
  • RAG · entity graph
  • long-term memory · permission filter

Proactive Intelligence
主动智能

  • cron · event triggers · webhooks
  • monitoring · notification queue
  • agent runner · action queue
  • approval policy
  • 每日 briefing / 风险提醒

L6 Architect 三大核心能力

System Resilience · 系统韧性

retry fallback structured output human escalation observability eval regression

Contextual Architecture · 上下文架构

  • Working memory — 任务状态、工具观察
  • Long-term memory — 偏好、规则、历史
  • Transactional memory — 审计、审批、状态变更

Orchestration & Integration · 编排与集成

  • multi-model routing
  • workflow vs agent 选型
  • single / multi-agent patterns
  • 并行 vs 串行 · 自动 vs 确认
  • agent ↔ human handoff

III · 实用判断框架（10 问）

1AI 只在开发时参与？→ AI-assisted building，非 AI 产品
2用户使用时 AI 参与？→ 进入 AI product 范畴
3一次性生成文本？→ Prompt Wrapper
4接入外部知识/业务上下文？→ Grounded AI / RAG
5能调用工具或 API？→ Tool-using AI
6嵌入稳定业务流程？→ LLM Workflow
7决定下一步、循环执行、观察调整？→ Agentic Core
8有 memory / permission / guardrails / eval / runtime？→ 接近 AI-native
9能主动触发（非等用户提问）？→ Proactive intelligence
10成为个人/组织工作系统一部分？→ L6 AI Architect 目标

人的能力四档 L3→L6

L3

The AI Consumer

Chatting · 把 AI 当咨询师

消费模型输出：提问 → 答案 → regenerate。上限取决于模型本身。

↔ 主要停留在 L1–L2 使用侧
L4

The AI Tinkerer / Vibe Coder

One-off · 把 AI 当外包

AI coding 做 demo/脚本/插件。Happy path 可行，edge cases 崩。

↔ 产出 L1 Prompt Wrapper / 普通 App
L5

The AI Builder

Reliability & Iteration · 把 AI 当逻辑引擎

Context Engineering · Evaluation · 迭代 workflow。职场需要稳定交付。

↔ 构建 L4 Workflow + L2/L3 组件
L6

The AI Architect

Orchestration & Integration

设计长期运行的 AI architecture：韧性、记忆、编排、主动系统。

↔ 设计 L5 Agentic + L6 AI-native System

能力跃迁 · 一句话

L3：我会问 AI。

L4：我会让 AI 帮我做一个东西。

L5：我会把 AI 放进一个可靠流程。

L6：我会设计一个 AI 驱动的系统。

终点：设计一个 AI 系统，让它在真实世界里可靠地替你工作。
